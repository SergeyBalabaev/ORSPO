# Task018 — Valgrind: поиск утечек памяти 🔍

## Что такое Valgrind?

**`Valgrind`** — это инструментальное программное обеспечение для отладки использования памяти, обнаружения утечек памяти и профилирования программ. Он эмулирует процессор и анализирует все обращения к памяти, что позволяет находить:

 * Утечки памяти (неосвобожденные блоки)

 * Использование неинициализированной памяти

 * Запись за границы выделенной памяти

 * Двойное освобождение памяти

 * Несоответствие вызовов malloc/free

 ## Полезные опции Valgrind

 | Опция |  Назначение |
| :--- | :--- |
|`--leak-check=full`|Полная проверка утечек памяти|
|`--show-leak-kinds=all`|Показать все типы утечек|
|`--track-origins=yes`|Отслеживать источник неинициализированных значений|
|`--verbose`|Подробный вывод|
|`--log-file=valgrind.log`|Сохранить вывод в файл|
|`--tool=memcheck`|Использовать Memcheck (по умолчанию)|


## Основные типы ошибок

 | Тип ошибки |  Описание |
| :--- | :--- |
|`Invalid write`|Запись за границы выделенной памяти|
|`Invalid read`|Чтение за границы выделенной памяти|
|`Conditional jump depends on uninitialized value`|Использование неинициализированной переменной|
|`definitely lost`|Утечка памяти (нет указателей на блок)|
|`indirectly lost`|Потеряны указатели на внутренние структуры|
|`still reachable`|Память не освобождена, но указатели существуют|

 ---

## Установка Valgrind
 
```bash
sudo apt-get install valgrind
```
 ---

 ## Компиляция и запуск с Valgrind

 ```bash
# Компиляция с отладочной информацией
gcc -g leak.c -o leak

# Запуск Valgrind для поиска утечек памяти
valgrind --leak-check=full ./leak

# Более подробный вывод
valgrind --leak-check=full --show-leak-kinds=all --track-origins=yes ./leak`
 ```
