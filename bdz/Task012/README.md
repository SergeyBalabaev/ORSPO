# Task012 — Основы Makefile 🔍 

## Что такое Makefile?
**Makefile** — это файл с инструкциями для утилиты make, которая автоматизирует сборку программ (например, компиляцию исходных файлов в исполняемый файл). Make отслеживает, какие файлы были изменены, и пересобирает только то, что действительно нужно.

---
## Основной синтаксис

```makefile
цель: зависимости
<TAB>команда1
<TAB>команда2
```
**Цель (target)** — задача, которую нужно выполнить (например, собрать программу).

**Зависимость (prerequisite)** — файлы, от которых зависит выполнение цели. Если зависимости изменились, цель будет выполнена заново.

**Команды**  действия, которые нужно выполнить для достижения цели. Важно: каждая команда должна начинаться с символа табуляции (TAB), а не с пробелов!

---
## Пример простого Makefile

```makefile
all: clean
	gcc -c *.c
	gcc *.o -o main

clean:
	rm -f *.o
```

В этом примере:

* Цель `all` зависит от цели `clean` (сначала выполнится очистка).

* Затем компилируются все `.c` файлы в объектные `.o` файлы.

* Из объектных файлов собирается исполняемый файл `main`.

* Цель `clean` удаляет все объектные файлы.

---

## Как запустить сборку

Для запуска сборки выполните в терминале:

```bash
make
```
По умолчанию утилита `make` ищет файл с именем `Makefile` или `makefile` и выполняет первую цель (обычно `all`).

Чтобы запустить конкретную цель, укажите её имя:

```bash
make all #собрать программу
make clean # очистить объектные файлы
```

---

## Зависимости

Если требуется, чтобы при вызове одной цели перед ней автоматически выполнялась другая (например, очистка старых бинарников перед компиляцией), используйте зависимости.

В примере выше цель all зависит от clean, поэтому при make all сначала сработает очистка.

1. Удаляются все `.o` файлы (`clean`)

2. Компилируются все `.c` файлы (`gcc -c *.c`)

3. Собирается исполняемый файл (`gcc *.o -o main`)
