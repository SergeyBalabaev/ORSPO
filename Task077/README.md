# Task077 — std::string 5️⃣

Необходимо реализовать тип данных `String`, представляющий строку с динамическим выделением памяти.

Тип должен содержать:

- указатель на символьный массив  
- текущую длину строки  
- ёмкость выделенной памяти  
- указатели на функции (методы), реализующие операции со строкой.

---

# Требования к структуре

Структура должна содержать поля:

```c
data — указатель на строку
size — текущая длина строки
capacity — размер выделенной памяти
```

Также структура должна содержать **указатели на методы**:

```c
append()
push_back()
clear()
length()
print()
destroy()
```


# Требуемые операции

## 1. Конструктор

Функция должна:

- выделить память
- скопировать начальную строку
- инициализировать методы

Прототип:

```c
String string_create(const char *init);
```

## 2. append

Добавляет строку в конец.

Пример:
```c
String s = string_create("Hello");

append(" world")

Результат:
Hello world
```

Прототип:
```c
void append(String *s, const char *str);
```

## 3. push_back

Добавляет один символ в конец строки.

Пример:
```c
push_back('!')

Результат:

Hello world!
```

## 4. clear

Очищает строку.

После выполнения:
```c
size = 0
data = ""
```

## 5. length

Возвращает текущую длину строки.

## 6. print

Выводит строку на экран.

## 7. destroy

Освобождает выделенную память.

# Дополнительное требование

Если при добавлении символов не хватает памяти, необходимо увеличить capacity и перераспределить память через realloc.
Рекомендуется увеличивать размер в 2 раза.


# Дополнительное задание (не входит в БДЗ, но рекомендуется попробовать)

Реализуйте механизм COW (Copy-On-Write). Его идея заключается в том, что при копировании строки данные не копируются, копия создается только при модификации строки. 

```c
String a = string_create("Hello");
String b = copy(a)
```
В примере выше обе строки пока ссылаются на один и тот же участок памяти. Если одна строка изменяется:
```c
append(b, " world")
```
то создаётся новый буфер, и строки становятся независимыми.
