# Task038: Inline-функции 3️⃣

## Задание

Вам дан файл `inlinetest.c`, который измеряет скорость выполнения **двух функций**:

1. `slow_func` — обычная функция с атрибутом `noinline`.
2. `fast_func` — объявлена как `static inline`.

Программа выполняет следующие действия:

- В цикле `N = 100_000_000` раз вызывает каждую функцию с одним и тем же значением.
- Измеряет время выполнения в тактах процессора с помощью `rdtsc()`.
- Измеряет время выполнения в микросекундах с помощью `clock_gettime(CLOCK_MONOTONIC_RAW)`.
- Выводит:
  - Время `slow_func` и `fast_func` в циклах и микросекундах.
  - Отношение скорости выполнения (`diff`).

**Пример компиляции и вызова программы:**

```bash
gcc -O2 -march=native inlinetest.c -o inlinetest && ./inlinetest
```
## Вопрос

- Почему fast_func работает намного быстрее, чем slow_func?
- Объясните влияние inline-функций на оптимизацию компилятора.
- Как это отражается на количестве команд и переходов в цикле?
